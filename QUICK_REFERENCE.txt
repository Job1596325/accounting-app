⚡ 快速开始（3步）
═══════════════════════════════════════════════════════════════════════════════

【第1步】启用 GitHub Pages
   1. 访问：https://github.com/Job1596325/accounting-app/settings
   2. 找到 "Pages" 部分
   3. Source 选择 "main" 分支，folder 选 "/ (root)"
   4. 点击 Save

【第2步】打开应用
   在微信或浏览器中打开：
   https://Job1596325.github.io/accounting-app/pro-optimized.html

【第3步】开始记账
   点击 "记一笔" 按钮开始！

⚙️ 数据存储原理
═══════════════════════════════════════════════════════════════════════════════

网页版：
  所有数据存储在浏览器的 LocalStorage 中
  └─ 自动保存
  └─ 永久存储（除非清除浏览器数据）
  └─ 容量：5-10MB

小程序版：
  所有数据存储在微信的本地存储中
  └─ 自动保存
  └─ 永久存储
  └─ 容量：10MB

🆘 问题排查
═══════════════════════════════════════════════════════════════════════════════

Q: 打不开网页版本？
A: 检查 GitHub Pages 是否已启用（见快速开始第1步）

Q: 数据丢失了？
A: 数据存储在 LocalStorage，清除浏览器数据会丢失
   建议：定期导出备份

Q: 小程序怎么发布？
A: 详见 MINIPROGRAM_GUIDE.md 的"发布小程序"部分

Q: 能在其他设备上使用数据吗？
A: 网页版：不同设备数据独立
   小程序：配置云开发可实现多设备同步
